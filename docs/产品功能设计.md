## 2. 各功能模块详细设计

### 2.1 老人档案

| 功能 | 说明 |
|------|------|
| 基础信息 | 姓名、性别、出生日期、身高、体重、血型、过敏史 |
| 疾病标签 | 为每位老人打上疾病标签（如：糖尿病、高血压、心脏病） |
| 紧急联系 | 紧急联系人、常用医院、主治医生 |
| 个人备注 | 自由文本备注区 |

### 2.2 血糖管理（母亲侧重点）

| 功能 | 说明 |
|------|------|
| 血糖记录 | 日期时间 + 数值(mmol/L) + 餐前/餐后 + 备注 |
| 预警阈值 | 可配置的阈值：空腹 ≥7.0 / 餐后 ≥10.0 触发预警 |
| 趋势图表 | 日/周/月趋势线图，标注正常范围区间 |
| 统计分析 | 平均值、标准差、达标率、最高/最低记录 |
| 快速记录 | 一键快速输入，移动端友好 |

### 2.3 心脏指标管理（父亲侧重点）

| 功能 | 说明 |
|------|------|
| 血压记录 | 收缩压 + 舒张压 + 心率 + 日期时间 + 备注 |
| 预警规则 | 收缩压≥140 或 ≤90，舒张压≥90 或 ≤60，心率≥100 或 ≤50 |
| 趋势分析 | 血压/心率趋势图，异常点标注 |
| 用药关联 | 关联当日服药情况，分析药物效果 |

### 2.4 用药管理

| 功能 | 说明 |
|------|------|
| 药品库 | 药名、规格、用途、副作用、存储方式、备注 |
| 用药计划 | 哪些药、每天几次、每次剂量、饭前/饭后 |
| 用药记录 | 实际服药打卡（已服/漏服/调整） |
| 配药记录 | 配药日期、医院、药品清单、下次配药提醒 |
| 药物交互 | 记录多药联用情况，标注已知交互风险 |

### 2.5 就诊管理

| 功能 | 说明 |
|------|------|
| 就诊记录 | 日期、医院、科室、医生、主诉、诊断、处方、医嘱 |
| 复诊提醒 | 下次复诊日期 + 提前N天提醒 |
| 就诊历史时间线 | 按时间倒序浏览所有就诊记录 |
| 检查报告 | 上传/记录检查报告关键数据 |

### 2.6 健康档案

| 功能 | 说明 |
|------|------|
| 病史记录 | 慢性病史、手术史、住院史、家族史 |
| 当前病痛 | 正在经历的不适/症状，持续时间，严重程度 |
| 健康时间线 | 综合所有健康事件的时间线视图 |
| 健康分析 | 基于数据的趋势分析和风险评估 |

### 2.7 医疗知识库 (Wiki)

| 功能 | 说明 |
|------|------|
| 知识条目 | Markdown 格式的知识文章 |
| 分类系统 | 多级分类 + 标签（疾病类型、器官系统、知识类型） |
| 关联功能 | 条目可关联到特定老人、特定疾病 |
| 全文搜索 | SQLite FTS5 全文搜索 |
| 来源标注 | 每条知识标注来源（医学指南/专家建议/新闻等） |
| 推荐阅读 | 医疗助手可推荐相关知识条目 |

**预设知识分类**:
- 糖尿病管理（饮食、运动、血糖控制、并发症预防）
- 高血压管理（生活方式、用药、监测）
- 心脏病管理（二尖瓣脱垂、心梗预防、猝死预防）
- 皮肤科（湿疹/皮炎护理、保湿、用药）
- 睡眠管理（失眠、安眠药使用）
- 用药常识（药物交互、服药注意事项）
- 饮食营养（低糖饮食、低盐饮食、心脏健康饮食）
- 急救知识（心梗急救、低血糖急救）

### 2.8 资讯中心

| 功能 | 说明 |
|------|------|
| RSS 采集 | 定时采集健康类 RSS 源（丁香医生、健康报等） |
| 本地存档 | 资讯内容保存到本地，防止链接失效 |
| 分类浏览 | 按疾病类型/资讯来源分类 |
| 收藏/分享 | 标记重要资讯，方便分享给老人 |
| 关键词监控 | 设置关键词（如"糖尿病新药"），自动高亮相关资讯 |

### 2.9 医疗助手

| 功能 | 说明 |
|------|------|
| 异常预警 | 血糖/血压超标时即时预警，推送到页面顶部 |
| 用药提醒 | 根据用药计划提醒今日用药 |
| 复诊提醒 | 复诊前N天提醒 |
| 健康建议 | 基于近期数据给出客观建议（如：血糖波动大建议就医） |
| 知识推荐 | 根据当前健康状况推荐相关知识文章 |
| 规则引擎 | 可配置的预警规则（阈值、频率、提醒方式） |

## 3. 数据库表结构设计 (SQLite)

### 3.1 核心表

```sql
-- 老人档案
CREATE TABLE persons (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    gender TEXT CHECK(gender IN ('M', 'F')),
    birth_date DATE,
    height_cm REAL,
    weight_kg REAL,
    blood_type TEXT,
    allergies TEXT,           -- 过敏史 (JSON array)
    emergency_contact TEXT,
    preferred_hospital TEXT,
    primary_doctor TEXT,
    disease_tags TEXT,        -- 疾病标签 (JSON array)
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 疾病/慢性病记录
CREATE TABLE conditions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    person_id INTEGER NOT NULL REFERENCES persons(id),
    name TEXT NOT NULL,           -- 如：2型糖尿病、高血压
    diagnosed_date DATE,
    diagnosed_by TEXT,            -- 诊断医生/医院
    severity TEXT CHECK(severity IN ('mild', 'moderate', 'severe')),
    status TEXT CHECK(status IN ('active', 'managed', 'resolved')) DEFAULT 'active',
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 健康指标表

```sql
-- 血糖记录
CREATE TABLE blood_sugar_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    person_id INTEGER NOT NULL REFERENCES persons(id),
    measured_at TIMESTAMP NOT NULL,
    value REAL NOT NULL,              -- mmol/L
    meal_context TEXT CHECK(meal_context IN ('fasting', 'before_meal', 'after_meal', 'bedtime', 'random')),
    notes TEXT,
    is_alert BOOLEAN DEFAULT FALSE,   -- 是否触发预警
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 血压/心率记录
CREATE TABLE blood_pressure_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    person_id INTEGER NOT NULL REFERENCES persons(id),
    measured_at TIMESTAMP NOT NULL,
    systolic INTEGER NOT NULL,        -- 收缩压
    diastolic INTEGER NOT NULL,       -- 舒张压
    heart_rate INTEGER,               -- 心率
    notes TEXT,
    is_alert BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 通用健康指标（可扩展）
CREATE TABLE health_metrics (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    person_id INTEGER NOT NULL REFERENCES persons(id),
    metric_type TEXT NOT NULL,        -- weight, temperature, spo2, etc.
    value REAL NOT NULL,
    unit TEXT NOT NULL,
    measured_at TIMESTAMP NOT NULL,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.3 用药管理表

```sql
-- 药品库
CREATE TABLE medications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    generic_name TEXT,            -- 通用名
    specification TEXT,           -- 规格
    category TEXT,                -- 分类
    usage_description TEXT,       -- 用途
    side_effects TEXT,            -- 副作用
    storage_notes TEXT,           -- 存储方式
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 用药计划
CREATE TABLE medication_plans (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    person_id INTEGER NOT NULL REFERENCES persons(id),
    medication_id INTEGER NOT NULL REFERENCES medications(id),
    dosage TEXT NOT NULL,             -- 剂量 (如: "1片")
    frequency TEXT NOT NULL,          -- 频率 (如: "每日3次")
    timing TEXT,                      -- 时机 (如: "饭后")
    start_date DATE NOT NULL,
    end_date DATE,
    is_active BOOLEAN DEFAULT TRUE,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 用药记录（打卡）
CREATE TABLE medication_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    plan_id INTEGER NOT NULL REFERENCES medication_plans(id),
    person_id INTEGER NOT NULL REFERENCES persons(id),
    taken_at TIMESTAMP NOT NULL,
    status TEXT CHECK(status IN ('taken', 'missed', 'adjusted', 'skipped')) DEFAULT 'taken',
    actual_dosage TEXT,               -- 实际剂量（如与计划不同）
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 配药记录
CREATE TABLE prescriptions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    person_id INTEGER NOT NULL REFERENCES persons(id),
    prescribed_date DATE NOT NULL,
    hospital TEXT,
    doctor TEXT,
    medications_json TEXT NOT NULL,    -- JSON: [{name, dosage, quantity, notes}]
    next_refill_date DATE,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.4 就诊管理表

```sql
-- 就诊记录
CREATE TABLE visits (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    person_id INTEGER NOT NULL REFERENCES persons(id),
    visit_date DATE NOT NULL,
    hospital TEXT,
    department TEXT,              -- 科室
    doctor TEXT,
    visit_type TEXT CHECK(visit_type IN ('initial', 'followup', 'emergency', 'checkup')),
    chief_complaint TEXT,         -- 主诉
    diagnosis TEXT,               -- 诊断
    prescription TEXT,            -- 处方
    doctor_advice TEXT,           -- 医嘱
    next_followup_date DATE,     -- 下次复诊日期
    attachments TEXT,             -- 附件路径 (JSON array)
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.5 知识库表

```sql
-- 知识分类
CREATE TABLE wiki_categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    slug TEXT NOT NULL UNIQUE,
    parent_id INTEGER REFERENCES wiki_categories(id),
    description TEXT,
    icon TEXT,
    sort_order INTEGER DEFAULT 0
);

-- 知识条目
CREATE TABLE wiki_articles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    slug TEXT NOT NULL UNIQUE,
    content TEXT NOT NULL,            -- Markdown
    category_id INTEGER REFERENCES wiki_categories(id),
    tags TEXT,                        -- JSON array
    source TEXT,                      -- 来源
    source_url TEXT,
    related_conditions TEXT,          -- 关联疾病 (JSON array)
    related_persons TEXT,             -- 关联老人 (JSON array)
    is_recommended BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- FTS5 全文搜索索引
CREATE VIRTUAL TABLE wiki_fts USING fts5(
    title, content, tags,
    content=wiki_articles,
    content_rowid=id
);
```

### 3.6 资讯表

```sql
-- 新闻资讯
CREATE TABLE news_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    url TEXT,
    source TEXT,                      -- 来源 (RSS feed name)
    content TEXT,                     -- 本地存档内容
    summary TEXT,
    published_at TIMESTAMP,
    category TEXT,
    tags TEXT,                        -- JSON array
    is_bookmarked BOOLEAN DEFAULT FALSE,
    is_shared BOOLEAN DEFAULT FALSE,  -- 是否已分享给老人
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- RSS 源配置
CREATE TABLE rss_feeds (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    url TEXT NOT NULL,
    category TEXT,
    is_active BOOLEAN DEFAULT TRUE,
    last_fetched_at TIMESTAMP,
    fetch_interval_hours INTEGER DEFAULT 4,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.7 系统表

```sql
-- 预警规则
CREATE TABLE alert_rules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    person_id INTEGER REFERENCES persons(id),  -- NULL = 全局规则
    rule_type TEXT NOT NULL,          -- blood_sugar_high, bp_high, etc.
    condition_json TEXT NOT NULL,     -- {"metric": "blood_sugar", "operator": ">=", "threshold": 7.0}
    message_template TEXT NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 预警记录
CREATE TABLE alert_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    rule_id INTEGER NOT NULL REFERENCES alert_rules(id),
    person_id INTEGER NOT NULL REFERENCES persons(id),
    triggered_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    metric_value REAL,
    message TEXT,
    is_acknowledged BOOLEAN DEFAULT FALSE,
    acknowledged_at TIMESTAMP
);

-- 操作日志
CREATE TABLE activity_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    person_id INTEGER REFERENCES persons(id),
    action TEXT NOT NULL,
    entity_type TEXT,
    entity_id INTEGER,
    details TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 系统配置
CREATE TABLE settings (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 4. 可扩展架构设计

### 4.1 插件化模块注册

```python
# 每个功能模块遵循统一接口
class HealthModule:
    name: str
    slug: str
    icon: str
    nav_order: int
    
    def get_routes(self) -> list[Route]
    def get_models(self) -> list[Model]
    def get_nav_items(self) -> list[NavItem]
    def get_dashboard_widgets(self, person_id: int) -> list[Widget]
```

### 4.2 新增模块流程

1. 在 `modules/` 下创建新模块目录
2. 实现 `HealthModule` 接口
3. 注册到模块注册表
4. 自动出现在导航、仪表盘

### 4.3 数据层扩展

- SQLAlchemy 模型自动发现
- Alembic 迁移管理表结构变更
- JSON 字段用于灵活扩展属性

## 5. 知识采集机制

### 5.1 采集源

| 源 | 类型 | 说明 |
|----|------|------|
| 丁香医生 | RSS | 疾病科普、用药指南 |
| 健康报 | RSS | 健康新闻 |
| PubMed | API | 医学研究摘要（英文） |
| 知网/万方 | 手动 | 中文医学文献 |
| 微信公众号 | 手动 | 健康类公众号文章（手动复制入库） |

### 5.2 采集流程

```
RSS Feed → feedparser 解析 → 去重 → 分类 → 存入 news_items
    ↓
关键资讯 → 自动关联到相关疾病/老人
    ↓
医疗助手 → 推荐给用户阅读/分享
```
