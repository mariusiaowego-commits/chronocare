## 云胶片影像查看器

### 问题
- 之前下载的 1512 张 CT 影像全部损坏（UTF-8 charset 解码问题）
- 服务器返回 `Content-Type: image/jpeg;charset=UTF-8`，Playwright `response.body()` 将二进制当 UTF-8 解码
- `ffd8ffe0` → `efbfbdefbfbdefbfbd`，14258 处替换不可逆

### 解决方案
1. **XHR 拦截 + UID 映射**：在浏览器内部捕获 `ArrayBuffer` 响应
2. **hierarchy API**：获取完整 DICOM 层级结构（Patient → Study → Series → Image）
3. **按 series 分类存储**：10 个影像序列，1377/1512 张有效图片
4. **前端展示**：series tab 切换、全屏查看、键盘导航

- 下载脚本：`scripts/download_ct_final.py`

## 学到的坑点
1. **Playwright charset 解码陷阱**：`response.body()` 在 `charset=UTF-8` Content-Type 下会将二进制数据当 UTF-8 解码
2. **XHR vs Fetch**：现代 SPA 可能用 Fetch API 而非 XMLHttpRequest，需同时拦截
3. **data: URL 只是缩略图**：页面上的 data: URL 是缩略图（~39KB），全分辨率图片通过 fetch/XHR 异步加载（~56-58KB）
