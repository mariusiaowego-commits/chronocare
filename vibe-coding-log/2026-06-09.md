## 今日三件事

### 1. 数据汇总分析(基于现有 DB)

**输入**:用户给出 3 个分析问题:
1. 过去看哪个医生更多?
2. 不同医生诊断是否一致?
3. 过去到现在的病情变化?

**产出**:完整分析报告 + 关键发现

**关键挑战与解决**:

|挑战 | 解决 |
|---|---|
| visits 表里 `doctor` 字段全 None(电子病历没存医生) | 从 `medical_records.ocr_text` 里反向提取 PDF 签名(2024-03 ~ 2025-02 这段有签名,2025-04 后改成图片格式)|
| 签名提取受 OCR 噪声干扰(处方药名混入候选) | 两轮策略: 第一轮黑名单过滤→锁定高频医生(>=2 次),第二轮白名单召回 → 7 个真医生 + 1 个 OCR 噪声 |
| 变量名复用导致 c 变成 int | `for d, c in high_conf.most_common():` → `for name, cnt:`, cursor 被覆盖 |
| 崔洁 5 次诊断只有"失眠"看起来像医生分歧 | 同日 2 张 PDF 配对分析 → 崔洁是失眠专病门诊与张培培同日配对 |
| 2025-04 后 27 张 PDF 无签名 | 明确标注"无法追溯",不假装统计 |

### 2. 信息图生成(3 次迭代 = 完整的设计-评审-修正闭环)

**目标**:把汇总数据生成可视化海报,适合家庭健康档案。

**迭代记录**:
- **v1 (morandi-journal + winding-roadmap)**: 视觉到位但术语太专业("二尖瓣关闭不全"吓人)
- **v2 (改成老人话)**: 完全去掉"中风/出血/倒漏",但矫枉过正 → "需手术评估"被改成"可以慢慢聊不着急" → 误导病人觉得自己没事
- **v3 (准确 + 温和平衡)**: 恢复医学术语 + 同时给口语解释(括号法),用色区分"稳定"(橄榄绿)vs"需进一步治疗"(陶土橘)

**关键学习**:**"温和"≠"淡化病情"**。让病人安心 ≠ 让病人小看病。一个好的健康物料要同时做到:
- 该说"中风风险"就说(医学必要)
- 但用"血的稀稠度偏高,有中风风险"而非"中风危险!!"(专业平和)
- 每个"需治疗"项都给出明确下一步动作
