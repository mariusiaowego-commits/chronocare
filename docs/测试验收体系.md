## 1. 测试策略

### 1.1 测试金字塔

```
        /  E2E (少量)  \      -- 关键流程走通
       / 集成测试 (中量) \     -- API + DB 联动
      / 单元测试 (大量)    \   -- 业务逻辑、模型验证
```

### 1.2 框架与工具

| 工具 | 用途 |
|------|------|
| pytest | 测试框架 |
| pytest-asyncio | 异步测试 |
| httpx + ASGITransport | FastAPI 集成测试 |
| pytest-cov | 覆盖率报告 |
| factory-boy | 测试数据工厂 |
| freezegun | 时间冻结 |

### 1.3 覆盖率要求

- 核心业务逻辑（services/）: >= 90%
- API 路由（routers/）: >= 80%
- 数据模型（models/）: >= 80%
- 总体: >= 75%

```python
# tests/test_services/test_blood_sugar.py

class TestBloodSugarService:
    def test_create_record_normal(self, db_session, mother):
        """正常血糖记录创建"""
        record = create_blood_sugar(db_session, person_id=mother.id,
            value=5.6, meal_context="fasting", measured_at=datetime.now())
        assert record.value == 5.6
        assert record.is_alert is False

    def test_create_record_triggers_alert(self, db_session, mother):
        """血糖超标触发预警"""
        record = create_blood_sugar(db_session, person_id=mother.id,
            value=8.5, meal_context="fasting", measured_at=datetime.now())
        assert record.is_alert is True
        # 检查预警日志
        alert = get_latest_alert(db_session, mother.id, "blood_sugar_high")
        assert alert is not None
        assert "8.5" in alert.message

    def test_fasting_threshold_7(self, db_session, mother):
        """空腹血糖阈值 7.0 mmol/L"""
        r1 = create_blood_sugar(db_session, mother.id, 6.9, "fasting")
        assert r1.is_alert is False
        r2 = create_blood_sugar(db_session, mother.id, 7.0, "fasting")
        assert r2.is_alert is True

    def test_postprandial_threshold_10(self, db_session, mother):
        """餐后血糖阈值 10.0 mmol/L"""
        r = create_blood_sugar(db_session, mother.id, 9.5, "after_meal")
        assert r.is_alert is False
        r2 = create_blood_sugar(db_session, mother.id, 10.0, "after_meal")
        assert r2.is_alert is True

    def test_statistics_calculation(self, db_session, mother):
        """统计计算正确性"""
        for v in [5.0, 6.0, 7.0, 8.0, 5.5]:
            create_blood_sugar(db_session, mother.id, v, "fasting")
        stats = get_blood_sugar_stats(db_session, mother.id, days=7)
        assert stats["avg"] == 6.3
        assert stats["max"] == 8.0
        assert stats["min"] == 5.0
        assert stats["count"] == 5
```

### 5.1 预警阈值矩阵

| 指标 | 正常范围 | 预警阈值 | 测试用例 |
|------|---------|---------|---------|
| 空腹血糖 | 3.9-6.1 | >= 7.0 | test_fasting_threshold_7 |
| 餐后血糖 | < 7.8 | >= 10.0 | test_postprandial_threshold_10 |

| 舒张压 | 60-89 | >= 90 或 <= 60 | test_high/low_diastolic |
| 心率 | 60-100 | >= 100 或 <= 50 | test_high/low_hr |

### 5.2 时间处理

- 所有时间存储为 UTC
- 显示时转换为 Asia/Shanghai (UTC+8)
- 测试中使用 freezegun 冻结时间

## 6. 验收流程

### 6.1 开发自测

每个功能完成后，开发者自测：
1. 运行 `pytest tests/ -v` 全部通过
2. 浏览器手动操作核心流程
3. 手机端验证关键页面

### 6.2 用户验收

1. 部署到本地供用户访问
2. 按 Checklist 逐项验收
3. 记录问题到 GitHub Issues
4. 修复后再次验收
